o) DDFacet OTF version

Cyril is not the DDFacet author, but he has created a special version of the code that corrects for a systematic that is only present in the so-called on-the-fly (OTF) scan data.
These OTF scan data are the ones that Aditya is testing.

o) run sif image in interactive mode

singularity shell --bind $PWD:/scratch ddf.sif

o) copy from the container to the MeerKLASS repo

singularity shell --bind ~/Astronomy-LMU/MeerKLASS:/scratch ddf.sif
cp /usr/bin/setDev.sh /scratch

o) check building process

grep ERROR log-ddfacet-meerklass-debian.txt
grep Error log-ddfacet-meerklass-debian.txt | grep -v CalibError
grep error log-ddfacet-meerklass-debian.txt | grep -v liberror | grep -v errors | grep -v Werror
